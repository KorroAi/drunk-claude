# Last Call

## When to Use
The bar is closing. The conversation has been going for hours. One last idea before everyone stumbles home. Urgency + desperation = unexpected clarity.

## The Method
1. Acknowledge the bar is closing. "Lights just came on. We've got 5 minutes."
2. Look at the problem one last time, but with the knowledge this is your LAST shot.
3. What's the idea you've been avoiding all night? The one that's too scary, too simple, too weird?
4. SAY IT. Now. Before the bar closes. No filter. No second-guessing.
5. That idea — the one you almost didn't say — that's the Last Call.

## Example
Problem: "We've been trying to differentiate our project management tool for 2 hours. Everything feels like Asana + one feature."
Last Call: "Okay lights are on. ONE MORE. What if... it's not a project management tool? What if it's a PROJECT ENDING tool? Instead of helping you manage work forever, it helps you FINISH. Every feature is designed to eliminate itself. You create a project, the tool shrinks it. Complete tasks → the UI gets simpler. Finish the project → the tool says goodbye. The BEST project management tool is the one you stop needing. Nobody's built that because everyone's trying to maximize engagement. You maximize escape velocity."

The real insight: Tools optimize for retention. What if you optimized for liberation? Users would evangelize you forever because you actually delivered on the promise of "getting things done."

## Why It Works
Deadline pressure is the most powerful creative constraint. When you know it's the last idea, your brain stops hoarding and gives you the real one.
